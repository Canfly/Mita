# Aviary Canon

Канон проекта фиксирует неизменяемые принципы. Он используется при спорных изменениях и в арбитраже.

## Канон
1) Marketplace-first - сделки и эскроу в центре.
2) Общение только внутри сделки - клетка сделки.
3) Event Registry - Interaction Ledger хранит только метаданные.
4) Артефакты сделки живут оффчейн - CID, шифрование, TTL.
5) Worm обеспечивает связь при деградации интернета.
6) Полицентрическая политика - через Guardian домены.
